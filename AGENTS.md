# AGENTS.md - 流程铁律

> **备注**: 放根目录 ✅ 必须
> 定义 PM/DEV/OPS 职责边界，AI 和人都必须遵守的流程铁律

本文件定义了项目中的职责边界和协作规则。

## 角色职责

### PM (项目经理)
- 需求管理与优先级排序
- 项目进度跟踪与状态更新
- 跨团队协调与沟通
- 验收标准制定

### DEV (开发)
- 代码实现与重构
- 单元测试编写
- 技术方案设计与评审
- 代码审查与合并

### OPS (运维)
- 部署与发布管理
- 监控与告警配置
- 故障响应与恢复
- 运维文档维护

## 工作流程

1. **需求阶段**: PM 创建需求文档 → DEV/OPS 评审 → 确定技术方案
2. **开发阶段**: DEV 领任务 → 实现 → 自测 → 提交 PR
3. **测试阶段**: OPS 部署测试环境 → QA 验证 → 修复问题
4. **发布阶段**: OPS 部署生产 → 监控验证 → 通知干系人

## 铁律

- 所有变更必须经过代码审查
- 重大决策必须记录在 `docs/decisions/`
- 部署前必须通过测试
- 生产环境变更必须记录在 `docs/09_runbook.md`
