# ADR-0001-项目模板创建

- 状态: 已通过
- 日期: 2026-02-08
- 决策人: doggo

## 背景

需要创建一个标准化的项目模板，用于统一所有项目的目录结构和工作流程。

## 决策

创建 `your-project` 仓库作为项目模板，包含以下核心文件:

- AGENTS.md - 流程铁律
- TOOLS.md - 工具规则
- SOUL.md - 项目人格
- README.md - 项目入口
- PROJECT.md - 项目总览
- docs/ - 完整文档体系

## 影响

### 正面影响

- 统一项目结构，便于维护
- 新项目可以快速启动
- AI 可以通过文档快速理解项目

### 负面影响

- 模板维护成本
- 灵活性受限

## 替代方案

1. 不使用模板 - 被否决（缺乏一致性）
2. 使用 GitHub 模板仓库 - 采用此方案

## 实施计划

1. 创建仓库
2. 初始化目录结构
3. 创建核心文档
4. 提交并推送

## 备注

模板应该随着项目实践不断迭代优化。
